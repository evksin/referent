# Чеклист тестирования приложения Referent

## Этап 6: Тестирование

### 1. Тестирование каждой кнопки с реальными URL статей

#### Кнопка "О чем статья?" (summary)
- [ ] Ввести URL англоязычной статьи
- [ ] Нажать кнопку "О чем статья?"
- [ ] Проверить отображение индикатора загрузки "Анализирую статью..."
- [ ] Дождаться результата
- [ ] Проверить, что результат содержит краткое описание (2-3 абзаца)
- [ ] Проверить, что результат на русском языке
- [ ] Проверить кнопку копирования результата

#### Кнопка "Тезисы" (theses)
- [ ] Ввести URL англоязычной статьи
- [ ] Нажать кнопку "Тезисы"
- [ ] Проверить отображение индикатора загрузки "Выделяю основные тезисы..."
- [ ] Дождаться результата
- [ ] Проверить, что результат представлен в виде маркированного списка
- [ ] Проверить, что тезисы краткие и информативные
- [ ] Проверить форматирование списка (символы •)
- [ ] Проверить кнопку копирования результата

#### Кнопка "Пост для Telegram" (telegram)
- [ ] Ввести URL англоязычной статьи
- [ ] Нажать кнопку "Пост для Telegram"
- [ ] Проверить отображение индикатора загрузки "Создаю пост..."
- [ ] Дождаться результата
- [ ] Проверить наличие эмодзи в посте
- [ ] Проверить наличие хештегов в конце поста
- [ ] Проверить читаемость и структуру поста
- [ ] Проверить кнопку копирования результата

### 2. Тестирование с разными типами статей

#### Новостные статьи
- [ ] Протестировать с URL новостной статьи (например, BBC, CNN)
- [ ] Проверить работу всех трех кнопок
- [ ] Проверить качество результатов

#### Блог-посты
- [ ] Протестировать с URL блог-поста (например, Medium, Dev.to)
- [ ] Проверить работу всех трех кнопок
- [ ] Проверить качество результатов

#### Технические статьи
- [ ] Протестировать с URL технической статьи (например, документация, технические блоги)
- [ ] Проверить работу всех трех кнопок
- [ ] Проверить качество результатов

#### Длинные статьи
- [ ] Протестировать с URL длинной статьи (>5000 слов)
- [ ] Проверить, что контент обрезается корректно
- [ ] Проверить, что таймаут не срабатывает преждевременно

---

### 3. Проверка обработки ошибок

#### Неверный URL
- [ ] Ввести несуществующий URL
- [ ] Нажать любую кнопку
- [ ] Проверить отображение понятного сообщения об ошибке
- [ ] Проверить, что приложение не падает

#### Некорректный формат URL
- [ ] Ввести текст вместо URL
- [ ] Нажать любую кнопку
- [ ] Проверить валидацию URL

#### Статья без контента
- [ ] Ввести URL страницы без текстового контента
- [ ] Нажать любую кнопку
- [ ] Проверить сообщение "Не удалось извлечь контент статьи"

#### Отсутствие API ключа
- [ ] Удалить или переименовать OPENROUTER_API_KEY в .env.local
- [ ] Нажать любую кнопку
- [ ] Проверить сообщение об отсутствии API ключа

#### Превышение лимита запросов (429)
- [ ] Отправить множество запросов подряд
- [ ] Проверить обработку ошибки 429
- [ ] Проверить понятное сообщение пользователю

#### Таймаут (504)
- [ ] Использовать очень длинную статью или медленное соединение
- [ ] Проверить обработку таймаута (2 минуты)
- [ ] Проверить сообщение о превышении времени ожидания

#### Пустой ответ от AI
- [ ] Проверить обработку случая, когда AI возвращает пустой ответ
- [ ] Проверить сообщение пользователю

---

### 4. Проверка производительности и времени ответа

#### Время ответа
- [ ] Измерить время от нажатия кнопки до получения результата
- [ ] Проверить, что время ответа разумное (< 2 минут для обычных статей)
- [ ] Проверить индикатор загрузки во время ожидания

#### Производительность парсинга
- [ ] Проверить скорость парсинга разных типов статей
- [ ] Проверить обработку больших HTML-страниц

#### Производительность AI обработки
- [ ] Проверить время обработки коротких статей (< 1000 слов)
- [ ] Проверить время обработки средних статей (1000-5000 слов)
- [ ] Проверить время обработки длинных статей (> 5000 слов)

### 5. Проверка форматирования результатов

#### Форматирование тезисов
- [ ] Проверить, что тезисы отображаются списком
- [ ] Проверить правильность символов маркировки (•)
- [ ] Проверить читаемость и структуру

#### Форматирование поста Telegram
- [ ] Проверить использование эмодзи
- [ ] Проверить наличие хештегов
- [ ] Проверить структуру и читаемость
- [ ] Проверить длину поста (не слишком длинный)

#### Форматирование описания статьи
- [ ] Проверить структуру (2-3 абзаца)
- [ ] Проверить читаемость
- [ ] Проверить отсутствие списков в описании

#### Копирование результатов
- [ ] Проверить кнопку копирования для каждого типа результата
- [ ] Проверить, что скопированный текст сохраняет форматирование
- [ ] Проверить визуальную обратную связь (иконка меняется на галочку)
